# Test Output Directory

This directory contains test reports generated by the Syntheverse blockchain test suite.

## Report Files

Each test run generates two files:

1. **Text Report** (`test_report_YYYY-MM-DDTHH-MM-SS-sssZ.txt`)
   - Human-readable test report
   - Includes execution summary, errors, transactions, and logs
   - Formatted for easy reading

2. **JSON Report** (`test_report_YYYY-MM-DDTHH-MM-SS-sssZ.json`)
   - Machine-readable test report
   - Contains structured data for programmatic analysis
   - Includes all metadata, transactions, errors, and logs

## Report Contents

### Text Report Sections:
- **Test Execution Summary**: Start/end time, duration, status
- **Errors**: Any errors encountered during execution
- **Transactions**: All blockchain transactions with hashes and details
- **Execution Logs**: Complete chronological log of test execution
- **Conclusions**: Success/failure analysis

### JSON Report Structure:
```json
{
  "metadata": {
    "startTime": "ISO timestamp",
    "endTime": "ISO timestamp",
    "duration": seconds,
    "status": "SUCCESS" | "FAILED"
  },
  "summary": {
    "totalErrors": number,
    "totalTransactions": number,
    "totalLogs": number
  },
  "errors": [...],
  "transactions": [...],
  "logs": [...]
}
```

## Running Tests

To generate a new test report:

```bash
# Make sure blockchain is running
npm run node

# In another terminal, deploy contracts
npm run deploy:local

# Run the test (generates reports automatically)
npx hardhat run blockchain/scripts/test_with_papers.js --network localhost
```

Reports are automatically saved to this directory with timestamps.

## Analyzing Reports

### Quick Status Check
```bash
grep "Test Status" test_outputs/test_report_*.txt
```

### Find Failed Tests
```bash
grep "FAILED" test_outputs/test_report_*.txt
```

### Extract Transaction Hashes
```bash
grep "Hash:" test_outputs/test_report_*.txt
```

### Parse JSON Reports
```bash
cat test_outputs/test_report_*.json | jq '.metadata.status'
```

## Cleanup

To remove old test reports:
```bash
rm test_outputs/test_report_*.txt
rm test_outputs/test_report_*.json
```

Or keep only the latest:
```bash
ls -t test_outputs/test_report_*.txt | tail -n +2 | xargs rm
ls -t test_outputs/test_report_*.json | tail -n +2 | xargs rm
```
